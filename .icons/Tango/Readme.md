   *** -symbolic Icons ***

   The Symbolic icons was started in /places for display smaller Folders in Konqueror Sidepaneel.
   And it should be kept that way:
   you shouldn't use -symbolic icons in other theme directories,
   as this will unnecessarily bloat the icon theme!

   So don't use -symbolic icons in other directories, but support them!
   BUT:
   If a theme builder or designer creates larger icons in their theme for -symbolic,
   it should be supported by the Window Manager.

   In the main theme of the Window Manager, you should avoid using -symbolic
   icons in directories other than /places, because in a normal multi-color icon,
   you don't need larger icons, since the icons themselves represent icons,
   i.e., small images that are automatically scaled if they are in SVG format.

   So, refrain from creating extra -symbolic icons, especially for black and white icons,
   as this makes no sense anyway and only creates unnecessary icons (and symlinks) in bulk.

  So, basically, keep the number of icons small!!!

  Above all:
   If you maintain a window manager icon set,
   keep the number of icons even smaller so that you give the designers
   the opportunity to create an icon theme individually.

   Otherwise, no one will create icon themes on their own,
   given the sheer number of icons that they can no longer manage.

   Above all, eliminate all -symbolic icons in
   every directory except /places ! It's simply not needed and is pure nonsense.

   So if you want new icon themes, support the designers by keeping
   the number of icons small and not using -symbolic icons except in /places,
   but supporting them in the WM for the designers.
   Most of it goes to Breeze (looking to Breeze)

So, support the designers and WM! (and not the whole Programs..)
